Стратегия миграции данных:

- Подключение нового микросервиса будет происходить через паттерн Anticorruption Layer. При подключении микросервиса данные будут маршрутизироваться из монолита в микросервис.
- Для историчности и целостности данных их необходимо смигрировать из монолита в микросервис.
- Для обратной историчности данные направляемые в микросервис так же необходимо сохранять в БД Монолита, чтобы при переключении потока запросов с микросервиса в монолит данные были консистентны.
- Изменения данных для синхронизации можно передавать через Kafka с применением Changing Data Capture.
